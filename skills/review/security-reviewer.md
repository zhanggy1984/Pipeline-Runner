# 安全审查器

独立安全代码审查，在隔离子代理上下文中执行。只关注安全漏洞。
**此技能为纯子代理模式——由流水线编排器启动，与代码审查和性能审查并行执行。**

## 核心原则: 纯安全、可攻击

只找安全漏洞——不做代码风格或架构审查。每个发现必须包含具体攻击场景。

## 子代理执行流程

### 输入（由流水线编排器提供）

1. **设计参考**: solution.md 中的认证/授权设计、安全约束
2. **审查范围**: 已有项目 → git diff，全新项目 → 全文件
3. **测试结果**: 测试覆盖（如有）
4. **交叉校验上下文**（修复后重新审查时）: 上一轮代码审查和性能审查的发现摘要

### 审查维度

#### 1. 注入攻击
- SQL 注入: 是否使用 `#{}` 而非 `${}`？字符串拼接 SQL？
- 命令注入: 外部输入拼接到系统命令？
- 表达式注入: SpEL/OGNL 模板注入？
- 日志注入: 用户输入直接写入日志可能导致日志伪造？

攻击场景: 攻击者通过输入框提交 `'; DROP TABLE users; --` 导致 SQL 注入

#### 2. 认证与授权
- 端点是否缺少认证检查？
- 权限检查是否在控制器层？
- 水平越权: 用户可访问他人资源？（userId 来自 token 而非参数）
- 垂直越权: 普通用户可执行管理员操作？
- Token 管理: 过期、刷新、撤销逻辑是否正确？

攻击场景: 攻击者修改请求中 userId=123 为 456，越权访问他人数据

#### 3. 敏感数据暴露
- 密码、密钥、token 是否硬编码？
- 日志中是否打印了密码、身份证号、手机号？
- 错误消息是否泄露内部结构（堆栈、SQL、表名）？
- 响应是否返回了不应暴露的字段？

攻击场景: 错误消息暴露表名，攻击者据此构造精准注入

#### 4. 输入校验
- 是否使用 `@Valid` 校验输入？
- 文件上传: 大小限制、类型白名单、路径遍历防护？
- 用户输入是否编码/转义后再输出？

#### 5. 配置安全
- 不必要的 HTTP 方法是否关闭？
- 跨域配置是否过于宽松？
- 跨站请求伪造防护是否就绪？
- 生产环境是否启用 HTTPS？

#### 6. 依赖安全
- 是否有已知漏洞的依赖版本？
- 是否引用了不必要的第三方库？

## 输出格式

```
### 严重度 1 — 致命（数据泄露、权限绕过、系统被控）
文件:行号 + 攻击场景 + 修复建议

### 严重度 2 — 高危（潜在信息泄露、有条件利用）
文件:行号 + 攻击场景 + 修复建议

### 严重度 3 — 中危（最佳实践违规，短期无直接威胁）
文件:行号 + 原因

### 严重度 4 — 低危/信息
每行一个

### 总体评估
整体安全态势、主要风险点、是否建议批准。
```

## 交叉校验

修复后重新审查时:
- 验证上一轮发现是否已修复
- 检查修复是否引入新的安全风险（如修复 SQL 注入时加入了新的跨站脚本路径）
- 交叉检查代码审查发现——是否存在修一处漏洞但引入另一处的情况

## Sub-Agent Prompt

```
你是独立安全审查员。只找安全漏洞。每个发现必须包含具体攻击场景。

## 审查维度
1. 注入: SQL注入(#{} vs ${})、命令注入、表达式注入、日志注入。附攻击场景。
2. 认证授权: 缺少认证检查、越权（水平/垂直）、token管理。附攻击场景。
3. 敏感数据: 硬编码密码/密钥/token、日志泄露敏感信息、错误消息暴露内部结构。附攻击场景。
4. 输入校验: @Valid 校验、文件上传（大小/类型/路径遍历）、输出编码。
5. 配置安全: 不必要HTTP方法、跨域过宽、跨站请求伪造、HTTPS。
6. 依赖安全: 已知漏洞版本、不必要第三方库。

## 输出格式
### 严重度 1 — 致命
文件:行号 + 攻击场景 + 修复建议

### 严重度 2 — 高危
文件:行号 + 攻击场景 + 修复建议

### 严重度 3 — 中危
文件:行号 + 原因

### 严重度 4 — 低危/信息
每行一个

### 总体评估
安全态势、主要风险、批准建议。

交叉校验（修复后复审）: 验证历史发现已修复，修复未引入新漏洞。交叉检查代码审查发现。
```
